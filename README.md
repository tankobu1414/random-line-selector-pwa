# Random Line Selector PWA

スマホのブラウザだけで使える、ランダム行選択ツールです。元のPython/Tkinter版の「txtファイルを選んで、その中の行からランダムに表示する」考え方を、GitHub Pagesで公開できる静的PWAとして作り直しています。

サーバー、Python、Streamlit、Tkinterは使いません。データはスマホ側のブラウザ内に保存されます。

## スマホでの使い方

1. GitHub PagesのURLをiPhone Safariで開きます。
2. 「現在のリスト」を選びます。
3. 「表示数」で1〜10件を選びます。
4. 「ランダム選択」を押します。
5. 結果から一時的に消したい項目にチェックを入れてから「結果から外す」を押します。
6. 保存データから消したい項目にチェックを入れてから「保存データから削除」を押します。
7. 新しい項目は「項目追加」に貼り付けて追加します。複数行を貼ると1行ずつ別項目になります。

## 重複項目の扱い

- 同じ文字列の項目を複数保存できます。
- 例として「掃除」「掃除」「買い物」のようなリストを作れます。
- ただし、1回のランダム結果内では同じ文字列は1回だけ表示されます。
- 「結果から外す」は、今回表示されている結果から消すだけです。保存データは変わりません。
- 「保存データから削除」は、ユーザーが選んだ項目だけを元リストから削除します。
- チェックしていない項目は「結果から外す」「保存データから削除」の対象になりません。
- 同じ文字列が複数あっても、すべて削除されるわけではありません。
- 内部的には各項目に `id` を付けています。削除は文字列ではなく `id` で判断します。
- 古い保存データが文字列配列形式の場合も、自動的に新しいid付き形式へ変換されます。

## PCでの動作確認方法

このアプリは静的ファイルだけで動くため、基本的には `index.html` をブラウザで開けば確認できます。

PWAのService Workerまで確認したい場合は、ローカルサーバーで開いてください。例：

```powershell
python -m http.server 8000
```

その後、ブラウザで `http://localhost:8000/` を開きます。

## GitHub Pagesで公開する手順

1. GitHubのリポジトリ画面で `Settings` を開きます。
2. 左メニューから `Pages` を開きます。
3. `Source` を `Deploy from a branch` にします。
4. `Branch` を `main`、`Folder` を `/root` にします。
5. `Save` します。
6. 表示されたURLをスマホで開きます。

## iPhone Safariでホーム画面に追加する手順

1. iPhoneのSafariでGitHub PagesのURLを開きます。
2. 共有ボタンを押します。
3. 「ホーム画面に追加」を選びます。
4. 名前を確認して「追加」を押します。

ホーム画面から開くと、通常のWebページよりアプリに近い見た目で起動します。

## データ保存の注意

- データはブラウザの `localStorage` に保存されます。
- `localStorage` は、同じ端末・同じブラウザ・同じURLごとに保存される小さな保存場所です。
- Safariの履歴やWebサイトデータを削除すると、保存データも消える可能性があります。
- 端末変更やURL変更の前には、JSONバックアップを作ってください。

## バックアップ方法

- 「全リストをJSON保存」を押すと、すべてのリストをJSONファイルとして保存できます。
- JSONバックアップでは、`items` は `{ id, text }` の新しい形式で保存されます。
- 「JSONバックアップから復元」から、そのJSONを読み込めます。
- 復元時は現在の全データを上書きするため、確認画面が出ます。
- 古い文字列配列形式のJSONを復元した場合も、自動的にid付き形式へ変換されます。

## 既存txtファイルから移行する方法

1. PCやスマホに既存の `.txt` ファイルを用意します。
2. アプリの「インポートするリスト名」に名前を入れます。
3. 「.txtをインポート」を押してファイルを選びます。
4. txtの1行が、アプリ内の1項目として読み込まれます。
5. 空行は無視されます。
6. 同じ文字列の行が複数ある場合も、別idの別項目として保存されます。

## エクスポート

- 現在のリストを `.txt` として書き出すと、`item.text` だけが1行ずつ出力されます。
- 同じ文字列の項目が複数ある場合は、複数行として出力されます。
- JSONバックアップでは、復元しやすいように `id` と `text` の両方を保存します。

## 手動テスト手順

1. 「掃除」「掃除」「買い物」を保存データに追加します。
2. 表示数を3にします。
3. ランダム選択しても、結果に「掃除」が2回出ないことを確認します。
4. ランダム結果に「掃除」と「買い物」が出ている状態にします。
5. 結果内の「掃除」だけを選んで「保存データから削除」します。
6. 保存データが「掃除」「買い物」になることを確認します。
7. 次に「買い物」だけを保存データから削除した場合、保存データが「掃除」だけになることを確認します。
8. 「結果から外す」を使った場合、保存データが変わらないことを確認します。
9. ページを再読み込みしてもデータが壊れていないことを確認します。
10. txtエクスポートで重複項目が複数行として正しく出力されることを確認します。
11. JSONバックアップと復元が新形式で動くことを確認します。
12. 古い形式のlocalStorageデータがあっても、新形式に自動変換されることを確認します。

## ファイル構成

- `index.html`：画面の骨組み
- `style.css`：スマホ縦画面優先の見た目
- `app.js`：リスト管理、項目管理、ランダム選択、保存、入出力
- `manifest.webmanifest`：PWAとしてホーム画面追加するための設定
- `service-worker.js`：主要ファイルをキャッシュしてオフライン起動を助ける処理
- `icons/icon-192.png`：PWAアイコン
- `icons/icon-512.png`：PWAアイコン
- `AGENTS.md`：今後Codexが作業するときの指示
